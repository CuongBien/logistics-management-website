# 🤖 Gemini GitHub Actions Integration Guide

This guide explains how to interact with Gemini within this repository and how to configure the necessary infrastructure for these workflows to function.

## 💬 Interacting with Gemini via Comments

The `gemini-dispatch.yml` workflow allows you to talk to Gemini directly on GitHub Issues or Pull Requests. It looks for comments that start with `@gemini-cli`.

### Common Commands

| Command | Action | Example |
| :--- | :--- | :--- |
| **Direct Prompt** | Triggers a general investigation or task. | `@gemini-cli can you explain why the Outbox pattern is used here?` |
| **`/review`** | Manually triggers a code review on a PR. | `@gemini-cli /review` |
| **`/triage`** | Manually triggers label analysis on an issue. | `@gemini-cli /triage` |
| **`/approve`** | Approves a plan generated by Gemini (for `plan-execute`). | `@gemini-cli /approve` |

> ⚠️ **Note:** For security, Gemini only responds to users with `OWNER`, `MEMBER`, or `COLLABORATOR` roles in this repository.

## ⚙️ Step 2: Configure GitHub Secrets

Workflows need authentication to access Gemini models. Go to **Settings > Secrets and variables > Actions > Secrets** in your GitHub repository.

### Option A: Using Google AI (Recommended for simplicity)
If you are using an API key from [Google AI Studio](https://aistudio.google.com/):
1.  **`GEMINI_API_KEY`**: Your Google AI API Key.

### Option B: Using Vertex AI (Google Cloud)
If you are using enterprise-grade Vertex AI:
1.  **`GOOGLE_API_KEY`**: (Optional) For certain Google services.
2.  **`SERVICE_ACCOUNT_EMAIL`**: The email of the GCP service account.
3.  **`GCP_WIF_PROVIDER`**: The Workload Identity Federation provider string.

---

## 📊 Step 3: Configure GitHub Variables

These are non-sensitive settings. Go to **Settings > Secrets and variables > Actions > Variables**.

| Variable | Recommended Value | Description |
| :--- | :--- | :--- |
| **`GEMINI_MODEL`** | `gemini-2.0-flash-exp` | The specific model version to use. |
| **`GOOGLE_CLOUD_PROJECT`** | *Your Project ID* | Required if using Vertex AI. |
| **`GOOGLE_CLOUD_LOCATION`** | `us-central1` | The GCP region for Vertex AI. |
| **`GEMINI_CLI_VERSION`** | `latest` | The version of the Gemini CLI to run. |
| **`GOOGLE_GENAI_USE_VERTEXAI`** | `false` | Set to `true` only if using Google Cloud Vertex AI. |
| **`UPLOAD_ARTIFACTS`** | `true` | Allows Gemini to upload logs/debug info for failed runs. |
